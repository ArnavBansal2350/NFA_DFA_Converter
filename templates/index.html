INTRO Understanding Finite Automata
NFA

Nondeterministic Finite Automaton

An NFA is a theoretical model of computation where for a given state and input symbol, the machine can transition to multiple states simultaneously or even no state at all.

  • Can have multiple transitions for same input
  • Can have empty (ε) transitions
  • Accepts if any path leads to final state
  • Easier to design for complex languages

Example: NFA for (a+b)*a

DFA

Deterministic Finite Automaton

A DFA is a finite automaton where for each state and input symbol, there is exactly one transition to a next state — no ambiguity, no choices.

  • Exactly one transition per state per symbol
  • No empty (ε) transitions allowed
  • Accepts if machine ends in a final state
  • Easier to implement in real systems

Equivalent DFA for (a+b)*a

⚡ Key Differences

🔀
Transitions: NFA can go to multiple states; DFA goes to exactly one state per input.
⚡
Determinism: NFA is nondeterministic (many choices); DFA is deterministic (one choice).
🏗️
Design: NFA is easier to design; DFA is easier to implement in hardware/software.
⚖️
Equivalence: Every NFA has an equivalent DFA that accepts the same language.
